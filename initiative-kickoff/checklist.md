# Initiative Kickoff Checklist

Use this checklist to ensure all components of a new initiative are properly set up.

## Pre-Creation Validation

- [ ] Goal statement is clear and actionable
- [ ] Timeline expectations are defined (explicit dates or default 2-week sprints)
- [ ] Priority level is determined (high for strategic, medium for operational)
- [ ] Key stakeholders/domains are identified
- [ ] No duplicate initiatives exist with similar goals

## Initiative Entity

- [ ] Title is action-oriented and includes timeline reference
- [ ] Description includes what, why, and key constraints
- [ ] Success criteria are quantifiable and measurable
- [ ] Priority is set appropriately

## Milestones (3-5 Required)

- [ ] Each milestone has a clear, measurable outcome
- [ ] Due dates are in YYYY-MM-DD format
- [ ] Milestones are sequential (earlier enables later)
- [ ] Timeline is realistic for scope
- [ ] Milestone pattern covers: Planning, Foundation, Development, Testing, Launch

## Workstreams

- [ ] Engineering workstream created (required for technical goals)
- [ ] Product workstream created (required for technical goals)
- [ ] Design workstream created (if UI/UX work needed)
- [ ] Marketing workstream created (if external communication needed)
- [ ] Sales workstream created (if revenue impact)
- [ ] Operations workstream created (if process changes needed)

## Initial Tasks

- [ ] 2-4 starter tasks created per workstream
- [ ] Tasks are linked to appropriate milestones
- [ ] Tasks have clear acceptance criteria
- [ ] Priority levels are assigned (high for blockers)
- [ ] No orphan tasks (all linked to workstream and initiative)

## Agent Assignments

- [ ] Each workstream has an assigned agent
- [ ] Agent type matches workstream domain
- [ ] Context includes initiative ID and responsibilities
- [ ] Orchestrator agent assigned for cross-functional coordination
- [ ] Agent autonomy configured if user specified preferences

## Launch Verification

- [ ] All entity IDs are captured for output tree
- [ ] User confirmation received before launch (if launching immediately)
- [ ] Initiative status set to active after launch
- [ ] Summary output includes complete initiative tree with all IDs

## Post-Kickoff Communication

- [ ] User provided with initiative ID
- [ ] All milestone IDs listed with due dates
- [ ] All workstream IDs listed with assigned agents
- [ ] Next steps clearly defined
- [ ] Expected first deliverables and timeline communicated

## Error Handling

- [ ] Failed creations reported with specific error
- [ ] Partial success handled (continue with remaining items)
- [ ] Corrective actions suggested for common issues
- [ ] Missing dependencies flagged (workspace context, etc.)
